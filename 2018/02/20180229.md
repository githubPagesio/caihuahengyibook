4. n+1 query是什么，可以怎样解决
答: n+1 query会影响网站效能，它经常发生在关联的model中，其中1笔用来查询一个model，而其他n笔都是查询相关的model资料
比如，有两个model:
Class has_many: students
Student belongs_to :class
那么当查询student.class.name时，第一个查询是关于user的，其他的查询都是关于class的
解决方法是使用includes
比如：
def index
@students = Student.includes(:class).all
end
这时后，通过使用student来遍历@students，并获得student.class.name就只需要用两次查询即可
